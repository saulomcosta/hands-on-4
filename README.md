# hands-on-4 docker compose up -d postgres rabbitmq redis prometheus grafana zipkin
Nesta fase, o sistema evolui para eventos de negócio, focando no domínio. Eventos como `OrderCreated`, `PaymentProcessed` e `EmailSent` representam ações reais, promovendo desacoplamento, clareza no fluxo e uma arquitetura orientada a eventos, onde serviços reagem de forma independente às mudanças.
